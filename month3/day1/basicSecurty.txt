🧩 بخش ۱ — مفاهیم پایه امنیت وب

قبل از کدنویسی، باید بدونی چی رو داری امن می‌کنی.

✅ Authentication (احراز هویت)

یعنی “آیا این شخص واقعا همون کسیه که ادعا می‌کنه؟”
مثال: وارد شدن به سایت با یوزرنیم/پسورد.

✅ Authorization (مجوز دسترسی)

یعنی “این شخص چه کارهایی مجازه انجام بده؟”
مثال: Admin می‌تونه محصولات رو حذف کنه، ولی User عادی نمی‌تونه.

✅ Session (نشست کاربر)

وقتی کاربر Login می‌کنه، سرور یک Session برایش می‌سازه.

این Session معمولا با یک Cookie نگه‌داری می‌شه.

مشکل: Session روی سرور ذخیره می‌شه → مقیاس‌پذیری سخت می‌شه. (برای همین JWT محبوبه).

✅ CSRF (Cross Site Request Forgery)

حمله‌ای که کاربر رو مجبور می‌کنه در سایتی دیگه عملیاتی انجام بده.

مثال: وارد سایت بانکی شدی، بعد روی لینکی کلیک کردی که از طرفت یک تراکنش اجرا می‌کنه.

راهکار: CSRF Token (توکن تصادفی در فرم‌ها).

✅ CORS (Cross-Origin Resource Sharing)

مرورگرها به صورت پیش‌فرض اجازه نمی‌دن یک سایت (frontend.com) به منابع یک سایت دیگه (api.com) دسترسی داشته باشه.

راهکار: تنظیم CORS در سرور برای اجازه به دامین‌های خاص.
